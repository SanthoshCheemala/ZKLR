# Zero-Knowledge Logistic Regression System

## Project Overview

This project implements a zero-knowledge proof system for logistic regression using PLONK proof system with BN254 curve. The implementation ensures privacy-preserving machine learning where model weights remain confidential while proving correct predictions.

## System Specifications

### Circuit Architecture
- **Total Constraints**: 170,668
- **Proof System**: PLONK with KZG polynomial commitment
- **Curve**: BN254 (128-bit security)
- **Components**:
  - Linear Layer: W*X + B (3 constraints)
  - Sigmoid Activation: Lookup table with 30,720 entries (170,665 constraints)

### Performance Metrics
- **Proof Size**: 584 bytes (constant regardless of input size)
- **Proving Time**: ~4 seconds per proof (laptop), faster on GPU
- **Verification Time**: ~1.5 milliseconds
- **Accuracy**: Meets 97% threshold on test dataset

### Implementation Features
- Parallel proof generation (80% CPU utilization)
- Circuit caching for improved performance
- Batch processing of 2000 samples
- Fixed-point arithmetic (Q32 for linear, Q16 for sigmoid)

## Running the System

### Build Production System
```bash
go build -o zkproof main.go
```

### Execute Proof Generation
```bash
./zkproof
```

This will:
1. Load or compile circuit (170,668 constraints)
2. Generate proving and verifying keys
3. Process 2000 student samples from dataset
4. Export metrics to data/cache/run_metrics.txt

### Security Demonstrations
```bash
# Build demo executable
go build -o security_demo cmd/security_demo/main.go cmd/security_demo/export.go

# Run client-server simulation
./security_demo -client-server

# Run specific security tests
./security_demo -correctness        # Valid proof verification
./security_demo -tampered-proof     # Proof substitution attack
./security_demo -tampered-input     # Public input tampering
```

### Generate Performance Metrics
```bash
cd metrics_output
python3 metrics_collect.py
```

This generates:
- Performance CSV files (proof size, timing, scalability)
- 7 visualization plots (PNG format)
- Metrics from actual system runs (not estimates)

## Technical Architecture

### Circuit Design
The system uses a recursive proof architecture where:
- Each sample generates a complete ZK proof
- Model weights (W, B) remain private
- Public inputs (X, Y) are verifiable
- Sigmoid activation implemented via log-derivative lookup table

### Fixed-Point Representation
- Linear computation: Q32 (32-bit fractional precision)
- Sigmoid input: Q10 (10-bit fractional)
- Sigmoid output: Q16 (16-bit fractional)
- Range handling: [-30, +30] with saturation

### Security Properties
1. **Correctness**: Valid proofs verify successfully
2. **Soundness**: Invalid proofs are rejected
3. **Zero-Knowledge**: No information leakage about private inputs
4. **Tamper-Resistance**: Any modification detected during verification

- Correct computation of sigmoid(W*X + B)
- Input-output relationship validity
- Adherence to 97% accuracy threshold
