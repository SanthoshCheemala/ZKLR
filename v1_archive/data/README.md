# Data Directory

Recommended layout:

- `raw/` — input CSVs and original datasets (read-only)
- `processed/` — normalized/feature-engineered data generated by scripts
- `models/` — trained model parameters and metadata (e.g., best_model_parameters.txt)
- `cache/` — compiled circuit caches, intermediate artifacts (`*.cache`)
